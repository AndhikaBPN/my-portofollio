2. NEXtrac
NEXtrac adalah sebuah project yang dimana digunakan untuk tracking license dari user yang membeli product perusahaan. di project ini aku ditugaskan untuk testing crud license dan mengetes apakah license yang di create bisa valid dan digunakan di product lain atau tidak. di project ini aku fokus ke manual testing berupa pembuatan testcase, testing ui, testing api, dan testing integration api nya

3. GROmart
Gromart adalah sebuah project mobile apps android e-commerce seperti shopee. Gromart sendiri adalah sebuah project yang dimana dipakai oleh Distributor untuk menjualkan barang-barangnya di sebuah platform (gromart) dan ada user yang bisa membeli nya seperti shopee.
di project ini aku fokus automation mobile tetapi juga membantu tim manual testing dalam membantu proses testing, berupa  pembuatan testcase, testing ui, testing api, dan testing integration api nya

4. Restrucuter Automation Project
Di fase ini, aku diminta oleh lead QA ku untuk membantu dia migrasi automation dari katalon ke selenium.
Jadi di kantor Paramadaksa Teknologi Nusantara (nexsoft) ini ada sebuah standarisasi component yang dibuat oleh tim UI/UX dan di implementasi oleh tim Front End dev. seperti component untuk select single date, date range, input, dropdown list, dropdown search, multi select dropdown, radio button, bahkan table yang ditampilkan di ui juga ada standarisasi nya.
aku ditugaskan untuk membuat seperti sebuah library agar component-component tersebut bisa dipakai di automation testing, jadi QA lain ketika ingin melakukan filter date range hanya perlu memanggil function yang aku buat tanpa perlu coding ulang logic filter date range nya.
dan juga karena ini migrasi dari katalon ke selenium, aku disuruh untuk membuat standarisasi struktur folder project yang akan digunakan di selenium dan juga cara-cara untuk menjalankan testnya.

5. GRObiz
Grobiz adalah sebuah project untuk membuat campaign dari principal, bisa berupa banner carousel, video, ataupun image. yang dimana nanti file-file tersebut akan di integrasikan ke gromart dan ditampilkan ke gromart. disini aku fokus menjalankan automation testing menggunakan selenium dan manual testing juga berupa pembuatan testcase, testing ui, testing api, dan testing integration api nya

1. PPOB
ppob adalah sebuah project yang dimana perusahaan ini sebagai aggregator untuk proses pembelian seperi pulsa, paket data, pln, air, bahkan voucher game. dimana di project ini aku fokus testing core di cms nya yang dimana digunakan untuk menampilkan produk produk yang akan dijual. lalu manual testing di bagian payment nya, dimana aku fokus untuk mengecek hasil dari payment apakah berhasil atau gagal, dan jika gagal aku juga mengecek keseluruhan case gagal nya, seperti nomor salah, produk habis, produk tidak aktif, nomor terblokir, dan lain lain.
ppob juga mempunyai versi pwa (webview) dan juga mobile apps (android dan ios). untuk pwa dan mobile apps digunakan oleh user untuk membeli produk produk yang ada di cms tanpa melalui merchant lain. disini aku juga fokus untuk manual testing seperti testing ui, api testing, integration testing, dan juga melakukan load test di cms nya

2. HRIS
hris adalah sebuah project internal milik tim HR yang digunakan untuk memantau data karyawan sekaligus absensi nya. di project ini aku testing untuk fitur absensi, fitur booking meeting room, dan juga fitur reporting. di project ini aku fokus di manual testing saja berupa testing ui, testing api, integration testing

3. LullaDream
lulladream adalah project yang menggunakan ai (elevenlabs). project ini adalah project untuk orang tua agar dapat menceritakan sebuah lullaby untuk anaknya tanpa harus capek ngomong karena di lulladream ini ada fitur voice nya, di project ini user bisa generate story sendiri atau menggunakan story yang telah disediakan oleh tim media. bisa menggunakan template voice di lulladream atau bahkan cloning voice mereka sendiri agar sang anak lebih merasa sedang dibacakan cerita oleh ibunya.
project ini berbasis pwa dan mobile apps (android). disini aku fokus untuk testing manual berupa ui testing, integration testing, payment testing untuk fitur manage plans dan topup coins

4. MinuteX
MinuteX adalah sebuah project untuk generate MoM. yang dimana cara kerjanya adalah dengan 3 fitur, yaitu 
1. bot join meeting seperti join di google meet, atau zoom meeting, lalu live transkrip meetingnya dan generate summary di akhir yang dimana summary itu adalah hasil MoM. 
2. audio record, seperti namanya jika meeting offline dan perlu mencatat maka fitur ini akan berguna karena user hanya perlu click record -> meeting -> stop record -> tunggu AI menyelesaikan tugasnya untuk membuat summary
3. audio upload, user hanya perlu upload audio meeting yang sebelumnya sudah sempat direkam dan tinggal menunggu ai untuk menghasilkan summary nya
di project ini aku fokus untuk manual testing seperti ui testing, integration testing, dan testing hasil ai nya apakah benar atau halu.
